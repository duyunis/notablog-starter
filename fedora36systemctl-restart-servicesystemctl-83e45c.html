Fedora36通过systemctl restart service启动失败之后程序通过systemctl再也起不来了

Posted on Fri, Sep 1, 2023 Linux

systemctl restart service启动程序失败之后通过systemctl start service也启动不了 执行systemctl status xxx.service 如果显示：

Sep 01 15:19:54 dyz-compiler systemd[1]: micflow-agent.service: Scheduled restart job, restart counter is at 5.
Sep 01 15:19:54 dyz-compiler systemd[1]: Stopped micflow-agent.service - Micflow Agent.
Sep 01 15:19:54 dyz-compiler systemd[1]: micflow-agent.service: Start request repeated too quickly.
Sep 01 15:19:54 dyz-compiler systemd[1]: micflow-agent.service: Failed with result 'exit-code'.
Sep 01 15:19:54 dyz-compiler systemd[1]: Failed to start micflow-agent.service - Micflow Agent.

关键信息：

  • Stopped xxx
  • Start request repeated too quickly.
  • Failed with result 'exit-code'.
  • Failed to start xxx

执行命令：

journalctl -xe | grep micflow-agent

如果有：

SELinux is preventing (ow-agent) from execute access on the file /usr/local/bin/micflow-agent.
                                                   /usr/local/bin/micflow-agent default label should be bin_t.
                                                   # /sbin/restorecon -v /usr/local/bin/micflow-agent
                                                   If you believe that (ow-agent) should be allowed execute access on the micflow-agent file by default.

执行：

sudo /sbin/restorecon -v /usr/local/bin/micflow-agent

这样一顿操作下来应该就解决了，接着start你的程序就可以了。
